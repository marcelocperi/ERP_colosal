Documentación Técnica

Arquitectura del Sistema

La plataforma está construida sobre una arquitectura Modular Monolítica utilizando el patrón MVC (Modelo-Vista-Controlador). Se organiza en módulos funcionales independientes encapsulados mediante Flask Blueprints.

Módulos Principales:
  • core: Gestión de autenticación, usuarios, roles y configuración base.
  • biblioteca: Lógica de negocio específica (libros, préstamos, inventario).

Stack Tecnológico

Componente Tecnología Descripción
Backend Python 3.9+ / Flask Framework web ligero y extensible.
Base de Datos MariaDB / MySQL Motor relacional transaccional.
Frontend HTML5 / Jinja2 / WinCSS Renderizado servidor con interfaz moderna estilo Windows 11.
Conectividad SMTP / REST API Servicios de correo y consumo de APIs externas.

Capas del Sistema

1. Capa de Presentación (Frontend/Views)

Utiliza plantillas Jinja2 con herencia (`base.html`). Los estilos se basan en una implementación personalizada de CSS (WinCSS) para ofrecer una experiencia de usuario similar a una aplicación nativa, con soporte para componentes interactivos (Modales, Tablas dinámicas).

2. Capa de Lógica de Negocio (Controllers/Routes)

Gestionada por los archivos routes.py en cada blueprint. Implementa:

  • Control de Acceso (RBAC): Decoradores @permission_required para validar permisos por recurso.
  • Transacciones: Manejo de atomicidad en movimientos de stock y préstamos.

3. Capa de Datos (Data Access)

Acceso directo mediante conector mariadb. Se utiliza un gestor de contexto get_db_cursor() para asegurar el cierre de conexiones y commits automáticos.

Servicios e Integraciones

Servicio de Correo (Email Service)

Módulo: services/email_service.py

Implementa envío asíncrono (Threading) mediante SMTP (Gmail). Funcionalidades:

  • Notificaciones Transaccionales: Préstamos, Devoluciones, Reservas.
  • Seguridad: Recuperación de claves, alertas de intentos fallidos.
  • Templating: Generación HTML dinámica con branding corporativo.
Datos Externos (DolarAPI)

Módulo: services/finance_service.py (integrado en lógica)

Captura diaria de cotización del dólar para cálculo de morosidad y valoración de activos.

  • Fuente: https://dolarapi.com/v1/dolares/oficial
  • Uso: Conversión de valores históricos y cálculo de multas.
